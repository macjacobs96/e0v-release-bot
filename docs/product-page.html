痛点与方案

以前 vs 现在

一个版本释放涉及 SDK、APK、健康检测三个模块，每个模块有 MR 链接和 Build 链接，还有 Release Note 和测试报告。以前靠人工催、手动整理，现在全自动。

😫 以前

  • 群聊里来回 @人催链接
  • 手动复制粘贴整理邮件
  • 容易漏模块、漏链接
  • 文件传来传去版本混乱
  • 发完还得手动归档
VS

😎 现在

  • 各负责人群里丢链接即可
  • 机器人自动收集、汇总
  • 进度一目了然，缺啥看得到
  • 文件拖进群自动识别分类
  • 说「发版」→「发送」，完事
使用流程

就 3 步

多人并行提交，发起人最后确认发送，跟群接龙一样简单。

1

群里丢链接

各模块负责人 @机器人，把 MR 和 Build 链接发到群里。支持文件拖拽上传 Release Note 和测试报告。

@机器人 SDK MR: ... @机器人 APK Build: ... @机器人 [上传文件]
2

预览确认

发起人 @机器人「发版」，机器人展示邮件内容预览。收件人可随时增删，不放心发「预览」看详情。

@机器人 发版 @机器人 预览 +收件人 xxx@email
3

发送

确认无误后回复「发送」，邮件即刻发出。状态自动清空，随时可以开始下一轮版本释放。

发送 📧 邮件已发送！ 自动清空状态
功能亮点

为什么好用

不是简单的「收到链接→发邮件」，背后有一整套智能处理逻辑。

🧠

智能模块识别

发「健康检测：」自动归到健康检测模块，不会被 URL 里的 Domain 干扰。SDK / APK 也同样自动识别。

👥

多人并行协作

SDK 负责人、APK 负责人、测试负责人各发各的，互不干扰。进度实时可见，谁没交一眼就知道。

📎

文件自动分类

拖个文件进群，机器人自动判断是 Release Note 还是测试报告，不需要手动标注。

📋

进度实时可见

随时发「进度」查看收集状态。SDK ✅ / APK ⬜ / 健康检测 ✅ / Release Note ⬜ — 差啥补啥。

📧

标准化邮件模板

三段式结构：版本链接 + Release Note + 测试报告。模块不发则自动标注，附件自动打包。

🔄

即用即走

发送后自动清空状态，下一轮接着用。收件人列表可精细管理，增删随时生效。
